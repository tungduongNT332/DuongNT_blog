## Tổng Quan

Dự án này nghiên cứu và so sánh hiệu năng của các thuật toán chữ ký số **hậu lượng tử (Post-Quantum Cryptography - PQC)** khi được tích hợp vào hệ thống Blockchain. Mục tiêu là đánh giá khả năng thay thế các thuật toán chữ ký số truyền thống (ECDSA, RSA) bằng các thuật toán kháng lượng tử trong bối cảnh máy tính lượng tử đang phát triển mạnh mẽ.

<Alert type="important">
Máy tính lượng tử có khả năng phá vỡ các hệ mật mã khóa công khai hiện tại (RSA, ECC) thông qua thuật toán Shor. Việc nghiên cứu PQC là cấp thiết để bảo vệ hệ thống blockchain trong tương lai.
</Alert>

## Các Thuật Toán Được So Sánh

### CRYSTALS-Dilithium

Dilithium là thuật toán chữ ký số dựa trên **lattice (lưới)**, được NIST chọn làm tiêu chuẩn chính cho chữ ký số hậu lượng tử (FIPS 204).

**Ưu điểm:**
- Tốc độ ký và xác minh nhanh
- Kích thước khóa và chữ ký hợp lý
- Được NIST tiêu chuẩn hóa (ML-DSA)

```python
# Ví dụ: Sinh khóa Dilithium
from dilithium import Dilithium2

public_key, secret_key = Dilithium2.keygen()
signature = Dilithium2.sign(secret_key, message)
is_valid = Dilithium2.verify(public_key, message, signature)
```

### FALCON

FALCON (Fast-Fourier Lattice-based Compact Signatures over NTRU) sử dụng cấu trúc lưới NTRU với kỹ thuật **GPV sampling**.

**Ưu điểm:**
- Chữ ký nhỏ gọn nhất trong các thuật toán lattice-based
- Phù hợp cho các hệ thống có băng thông hạn chế

```python
# Ví dụ: Ký số với FALCON
from falcon import SecretKey, PublicKey

sk = SecretKey(512)
pk = PublicKey(sk)
signature = sk.sign(message)
is_valid = pk.verify(message, signature)
```

### SPHINCS+

SPHINCS+ là thuật toán chữ ký số dựa trên **hash**, không phụ thuộc vào bất kỳ giả định toán học nào ngoài tính an toàn của hàm hash.

**Ưu điểm:**
- Bảo mật dựa trên giả định tối thiểu (chỉ cần hash an toàn)
- Không bị ảnh hưởng bởi các tiến bộ trong lý thuyết lưới

**Nhược điểm:**
- Chữ ký rất lớn (lên tới ~49KB)
- Tốc độ ký chậm hơn

## Tích Hợp Blockchain

Dự án triển khai một blockchain đơn giản bằng Python, trong đó mỗi transaction được ký bằng một trong ba thuật toán PQC:

```python
class Transaction:
    def __init__(self, sender, receiver, amount, pqc_algo="dilithium"):
        self.sender = sender
        self.receiver = receiver
        self.amount = amount
        self.pqc_algo = pqc_algo
        self.signature = None

    def sign(self, private_key):
        """Ký transaction bằng thuật toán PQC đã chọn"""
        data = self.calculate_hash()
        if self.pqc_algo == "dilithium":
            self.signature = dilithium_sign(private_key, data)
        elif self.pqc_algo == "falcon":
            self.signature = falcon_sign(private_key, data)
        elif self.pqc_algo == "sphincs":
            self.signature = sphincs_sign(private_key, data)
```

## Kết Luận

- **Dilithium** là lựa chọn tối ưu cho blockchain nhờ cân bằng giữa tốc độ và kích thước
- **FALCON** phù hợp khi cần chữ ký nhỏ gọn (IoT, embedded systems)
- **SPHINCS+** phù hợp cho các hệ thống yêu cầu bảo mật tối đa, không phụ thuộc giả định toán học
