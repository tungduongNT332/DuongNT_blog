## What is Burp Suite?

Burp Suite is an **integrated platform** for performing security testing of web applications. Developed by PortSwigger, it is the industry-standard tool for web application penetration testing.

The **Community Edition** (free) includes essential tools:
- **Proxy** — Intercept and modify HTTP/HTTPS traffic
- **Repeater** — Manually modify and resend requests
- **Decoder** — Encode/decode data in various formats
- **Comparer** — Visual diff between responses
- **Sequencer** — Analyze token randomness

<Alert type="note">
The **Professional Edition** adds active scanning, advanced Intruder attacks, and saved project files. For learning, the Community Edition is more than sufficient.
</Alert>

## Setting Up the Proxy

### Step 1: Configure Browser Proxy

Set your browser's proxy settings to:
- **Host:** `127.0.0.1`
- **Port:** `8080`

Or use Burp's built-in **Chromium browser** (recommended for beginners):

```
Proxy tab → Open browser
```

### Step 2: Install Burp's CA Certificate

To intercept HTTPS traffic, you need to install Burp's SSL certificate:

1. With Burp running, navigate to `http://burpsuite`
2. Download the CA certificate
3. Install it in your browser's certificate store as a trusted root CA

### Step 3: Intercept Traffic

```
Proxy → Intercept → Intercept is on
```

Now browse any website — the request will be **paused** in Burp for you to inspect and modify before forwarding.

## The Repeater (Most Used Tool)

Repeater lets you **manually modify and resend** individual HTTP requests:

1. Right-click any request in Proxy History → **Send to Repeater**
2. Modify headers, parameters, or body
3. Click **Send** to see the response
4. Compare responses between different payloads

### Example: Testing for SQLi

```http
GET /api/user?id=1' OR 1=1 -- HTTP/1.1
Host: target.com
Cookie: session=abc123
```

Modify the `id` parameter and observe response changes.

## The Intruder (Automated Attacks)

Intruder automates customized attacks by **fuzzing** specific positions:

### Attack Types

| Type | Description | Use Case |
|------|-------------|----------|
| **Sniper** | Single payload set, one position at a time | Testing each parameter individually |
| **Battering Ram** | Same payload in all positions simultaneously | Credential testing with same value |
| **Pitchfork** | Different payload sets, matched by index | Username:password pairs |
| **Cluster Bomb** | All combinations of all payload sets | Brute-force testing |

### Setting Up an Attack

1. Send a request to Intruder
2. Mark payload positions with `§` markers:
```http
POST /login HTTP/1.1
Host: target.com

username=§admin§&password=§password§
```
3. Load your payload lists
4. Start attack and analyze response codes/lengths

<Alert type="tip">
In Community Edition, Intruder attacks are **rate-limited**. For faster testing, consider writing a custom Python script using the `requests` library.
</Alert>

## Essential Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| `Ctrl+R` | Send to Repeater |
| `Ctrl+I` | Send to Intruder |
| `Ctrl+Shift+D` | Send to Decoder |
| `Ctrl+F` | Forward intercepted request |
| `Ctrl+D` | Drop intercepted request |

## Pro Tips

1. **Scope filtering:** Define target scope to filter out noise from third-party domains
2. **Match & Replace:** Auto-modify requests/responses using rules in Proxy Options
3. **Collaborator:** Use for out-of-band (OOB) vulnerability detection
4. **Extensions:** Install from BApp Store — **Logger++**, **Autorize**, and **JSON Beautifier** are essentials
