## 1. Tổng Quan Dự Án

Trong tương lai gần, máy tính lượng tử có khả năng bẻ khóa các hệ mật mã khóa công khai hiện tại (như RSA, ECC) thông qua thuật toán Shor. Việc chuyển đổi sang **Mật mã Hậu Lượng Tử (Post-Quantum Cryptography - PQC)** là cấp thiết để bảo vệ hệ thống Blockchain.

Tuy nhiên, rào cản lớn nhất khi đưa PQC lên Blockchain (đặc biệt là EVM) chính là **kích thước của khóa và chữ ký quá lớn**. Việc lưu trữ trực tiếp (on-chain) sẽ tiêu tốn lượng Gas khổng lồ và làm phình to kích thước chuỗi khối một cách phi thực tế.

Dự án này là một bộ công cụ hoàn chỉnh để thử nghiệm, đánh giá và so sánh các thuật toán PQC trên Blockchain Ethereum. Mục tiêu chính là xây dựng một **kiến trúc lưu trữ lai (Hybrid Storage)** nhằm giải quyết triệt để bài toán kích thước, kết hợp giữa lưu trữ ngoài chuỗi (off-chain) và xác minh trên chuỗi (on-chain).

## 2. Giải Pháp: Hybrid Proof Flow

Dự án sử dụng mô hình **Lưu trữ ngoài chuỗi - Xác thực toàn vẹn trên chuỗi**, bao gồm 4 bước hoạt động chính:

1. **Người gửi (Sender):** Tiến hành ký dữ liệu bằng thuật toán chữ ký số PQC (hoặc tạo khóa KEM đối với Kyber).
2. **Lưu trữ ngoài chuỗi (Off-chain Storage):** Khóa công khai (Public Key) và Chữ ký PQC (Signature) có dung lượng lớn được đẩy lên hệ thống lưu trữ ngoài chuỗi (như IPFS hoặc local storage).
3. **Lưu trữ trên chuỗi (On-chain Storage):** Người gửi tạo một giao dịch Smart Contract với chi phí cực thấp (sử dụng chữ ký ECDSA truyền thống để xác thực giao dịch). Smart Contract chỉ làm nhiệm vụ lưu trữ **mã băm (Hash) của dữ liệu**, **`ipfsCid`** trỏ tới chữ ký PQC off-chain, và tên **thuật toán** được sử dụng.
4. **Người xác minh (Verifier):** Lấy dữ liệu bản ghi từ Smart Contract, dùng `CID` để tải chữ ký và khóa công khai từ IPFS về, sau đó tự chạy kiểm tra tính toàn vẹn (bằng cách so sánh Hash) và xác minh tính hợp lệ của chữ ký PQC.

```text
│   └── kyber-kem/                # Thuật toán ML-KEM
└── .env.example                  # File cấu hình môi trường
```

## 4. Các Thuật Toán PQC Được Tích Hợp

Dự án triển khai và benchmark 4 thuật toán PQC hàng đầu, được NIST tiêu chuẩn hóa:

| Thuật toán | Loại | Tiêu chuẩn NIST | Điểm nổi bật trong dự án |
|---|---|---|---|
| **CRYSTALS-Dilithium** | Chữ ký số | FIPS 204 (ML-DSA) | Được dùng làm baseline chuẩn vì độ cân bằng cực tốt giữa tốc độ và kích thước. |
| **FALCON** | Chữ ký số | FIPS 206 (dự kiến) | Tối ưu hóa kích thước chữ ký nhỏ gọn nhất, cực kỳ phù hợp cho môi trường Blockchain. |
| **SPHINCS+** | Chữ ký số | FIPS 205 (SLH-DSA) | Đánh đổi kích thước chữ ký siêu lớn lấy độ bảo mật cực cao (dựa trên Hash). |
| **CRYSTALS-Kyber** | KEM | FIPS 203 (ML-KEM) | Không phải chữ ký số, được thiết kế riêng để trao đổi và đóng gói khóa mã hóa bí mật. |

## 5. Smart Contract

Smart Contract `Demo.sol` được viết bằng Solidity. Khác với cách làm thông thường, nó không trực tiếp lưu chữ ký mà chỉ đóng vai trò như một sổ cái bất biến lưu lại bằng chứng toàn vẹn:

```solidity
struct Record {
    uint256 recordId;
    address owner;
    bytes payload;
    bytes32 payloadHash;       // Mã băm kiểm tra tính toàn vẹn
    bytes32 pqcProofHash;      // Bằng chứng PQC (Hash của Payload + Chữ ký PQC)
    uint256 appNonce;          // Chống Replay Attack
    uint256 timestamp;
    bool encrypted;
    string mode;
    string ipfsCid;            // Mã định danh IPFS để tải lại Chữ ký off-chain
    string algorithm;          // Thuật toán PQC (VD: ml_dsa_44)
}
```

## 6. Hướng Dẫn Cài Đặt & Trải Nghiệm

**Bước 1: Cài đặt dependencies**
Dự án yêu cầu Python 3.8 trở lên.
```bash
git clone https://github.com/tungduongNT332/PQC-on-Blockchain.git
cd PQC-on-BlockChain
pip install -r shared/requirements.txt
```

**Bước 2: Cấu hình môi trường**
Tạo file `.env` từ `.env.example` và thiết lập các thông số:
```bash
RPC_URL=https://sepolia.infura.io/v3/YOUR_API_KEY
PRIVATE_KEY=your_private_key_here  # Khuyến cáo chỉ dùng ví Testnet
CONTRACT_ADDRESS=your_deployed_contract_address
```
*Lưu ý: Mọi thư mục thuật toán tự động đọc cấu hình chung này.* Nếu muốn kết nối với IPFS thực thay vì mô phỏng local, hãy thiết lập thêm `IPFS_API_URL`.

**Bước 3: Chạy thử thuật toán (Ví dụ với Dilithium)**
```bash
cd algorithms/dilithium

# Chạy giao dịch hybrid PQC + ECDSA
python scripts/pqc_demo.py

# Xác minh lại tính toàn vẹn bằng cách lấy dữ liệu từ On-chain & Off-chain
python scripts/verify_e2e.py --record-id <ID> --public-key-cid <CID>
```
