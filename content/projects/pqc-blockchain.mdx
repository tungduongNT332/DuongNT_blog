## Tổng Quan

Dự án xây dựng một bộ công cụ benchmark toàn diện để đánh giá khả năng tích hợp các thuật toán mật mã hậu lượng tử (Post-Quantum Cryptography - PQC) được NIST tiêu chuẩn hóa vào hạ tầng Blockchain Ethereum. Hệ thống đo lường hiệu năng trên cả **3 trụ cột an toàn thông tin**: Tính toàn vẹn (Integrity), Tính bảo mật (Confidentiality), và Tính sẵn sàng (Availability).

<Alert type="important">
Máy tính lượng tử có khả năng phá vỡ các hệ mật mã khóa công khai hiện tại (RSA, ECC) thông qua thuật toán Shor. Việc nghiên cứu PQC là cấp thiết để bảo vệ hệ thống blockchain trong tương lai.
</Alert>

## Các Thuật Toán Được So Sánh

| Thuật toán | Loại | Tiêu chuẩn NIST | Cơ sở toán học |
|---|---|---|---|
| **CRYSTALS-Dilithium** | Chữ ký số | FIPS 204 (ML-DSA) | Lattice (Module-LWE) |
| **FALCON** | Chữ ký số | FIPS 206 (dự kiến) | Lattice (NTRU + GPV) |
| **SPHINCS+** | Chữ ký số | FIPS 205 (SLH-DSA) | Hash-based |
| **CRYSTALS-Kyber** | Đóng gói khóa (KEM) | FIPS 203 (ML-KEM) | Lattice (Module-LWE) |

### CRYSTALS-Dilithium (ML-DSA-44)

Thuật toán chữ ký số dựa trên **lattice**, được NIST chọn làm tiêu chuẩn chính (FIPS 204). Trong dự án, Dilithium được sử dụng ở chế độ hybrid: PQC signature lưu off-chain, proof hash ghi on-chain.

```python
from pqcrypto.sign import ml_dsa_44

public_key, secret_key = ml_dsa_44.generate_keypair()
signature = ml_dsa_44.sign(secret_key, payload)
verified = ml_dsa_44.verify(public_key, payload, signature)
```

### FALCON (Falcon-512)

Sử dụng cấu trúc lưới NTRU với kỹ thuật **GPV sampling**. Cho ra chữ ký nhỏ gọn nhất trong các thuật toán lattice-based, phù hợp cho hệ thống có băng thông hạn chế.

### SPHINCS+ (SHA2-128f)

Thuật toán chữ ký dựa trên **hash**, không phụ thuộc vào giả định toán học ngoài tính an toàn của hàm hash. Chữ ký lớn hơn đáng kể nhưng có tính bảo mật cao nhất.

### CRYSTALS-Kyber (ML-KEM-512)

Thuật toán **đóng gói khóa (KEM)** dùng cho bảo mật dữ liệu. Trong dự án, Kyber được sử dụng để đóng gói khóa AES-256, mã hóa payload trước khi ghi on-chain.

## Tính Năng Chính

### Tính Toàn Vẹn (Integrity)

- **Traditional Mode**: Giao dịch ký bằng ECDSA tiêu chuẩn của Ethereum.
- **PQC Hybrid Mode**: Giao dịch ký kép — ECDSA (on-chain) + PQC signature (off-chain via IPFS/local). Smart contract lưu `pqcProofHash`, `ipfsCid`, và `algorithm`.
- **End-to-End Verification**: Script `verify_e2e.py` đọc record on-chain, tải chữ ký + public key từ off-chain, xác minh cả proof hash lẫn PQC signature.
- **Integrity Cases**: Script `integrity_cases.py` kiểm thử các tình huống giả mạo (tampered payload, wrong key, replay attack).

### Tính Bảo Mật (Confidentiality)

- **Signature Packs** (Dilithium, Falcon, SPHINCS+): Mã hóa payload bằng **AES-256-GCM** trước khi ghi on-chain, kết hợp PQC signature.
- **KEM Pack** (Kyber): Sử dụng **ML-KEM-512** để đóng gói khóa AES, mã hóa payload, rồi ghi on-chain.

### Tính Sẵn Sàng (Availability)

- Benchmark tự động chạy N giao dịch liên tiếp trên cả 3 mode.
- Xuất kết quả JSON và biểu đồ so sánh bằng matplotlib.

## Smart Contract

Contract `Demo.sol` (Solidity ^0.8.20) được deploy trên Sepolia testnet, mỗi bản ghi on-chain gồm:

```solidity
struct Record {
    uint256 recordId;
    address owner;
    bytes payload;
    bytes32 payloadHash;
    bytes32 pqcProofHash;
    uint256 appNonce;
    uint256 timestamp;
    bool encrypted;
    string mode;
    string ipfsCid;
    string algorithm;
}
```

- `payloadHash` — hash kiểm tra tính toàn vẹn dữ liệu
- `pqcProofHash` — hash bằng chứng PQC (`keccak256(payload + pqc_signature)`)
- `appNonce` — chống replay attack
- `ipfsCid` + `algorithm` — tham chiếu off-chain và thuật toán PQC

## Quy Trình Vận Hành

```bash
# 1. Cài đặt dependencies
pip install -r shared/requirements.txt

# 2. Cấu hình
cp .env.example .env
# Điền: RPC_URL, PRIVATE_KEY, CONTRACT_ADDRESS

# 3. Chạy demo (ví dụ: Dilithium)
cd algorithms/dilithium
python scripts/traditional_demo.py           # ECDSA truyền thống
python scripts/pqc_demo.py                   # Hybrid PQC + ECDSA
python scripts/verify_e2e.py --record-id <ID> --public-key-cid <CID>
python scripts/integrity_cases.py            # Kiểm thử toàn vẹn
python scripts/confidentiality_demo.py       # Mã hóa AES-256-GCM + PQC

# 4. Benchmark
python scripts/availability_benchmark.py --mode traditional --count 5
python scripts/availability_benchmark.py --mode pqc_hybrid --count 5
python scripts/availability_benchmark.py --mode pqc_confidential --count 5

# 5. So sánh kết quả
python benchmark/compare.py
python benchmark/compare_availability.py
```

## Kết Luận

- **Dilithium (ML-DSA)** — Lựa chọn tối ưu cho blockchain nhờ cân bằng giữa tốc độ ký/xác minh và kích thước chữ ký
- **FALCON** — Phù hợp khi cần chữ ký nhỏ gọn (IoT, embedded systems, băng thông hạn chế)
- **SPHINCS+** — Phù hợp cho hệ thống yêu cầu bảo mật tối đa, không phụ thuộc giả định lưới
- **Kyber (ML-KEM)** — Giải pháp đóng gói khóa hậu lượng tử cho bảo mật dữ liệu on-chain
